= Scraper projekt
:toc: left
:toclevels: 3
:sectnums:
:source-highlighter: rouge

Ez a projekt több, különálló scraper scriptet tartalmaz. A futtatáshoz ajánlott külön Python virtuális környezetet használni.

== Telepítés

A függőségeket ajánlott `.venv` virtuális környezetbe telepíteni:

[source,bash]
----
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
----

Windows alatt az aktiválás:

[source,powershell]
----
.venv\Scripts\activate
----

== Általános futtatási megjegyzések

A scraperek általában a `--output` paraméterrel megadott mappába mentik az eredményeket. Hosszabb futásnál érdemes külön kimeneti mappát használni minden scraperhez, hogy az adatok és az állapotfájlok áttekinthetőek maradjanak.

Hibakereséshez több scraper támogatja a `--headed` kapcsolót, amely látható böngészőablakban futtatja a Playwrightot.

== Scraperek

=== Agroinform scraper

A `agroinform_scraper.py` az `agroinform.hu` fórum témáit és hozzászólásait gyűjti ki, majd JSON formátumban menti el a megadott kimeneti mappába.

==== Futtatás

A legfontosabb futtatóparancs:

[source,bash]
----
python agroinform_scraper.py --output ./agro
----

A fenti parancs az eredményeket az `./agro` mappába menti.

==== Opcionális paraméterek

[source,bash]
----
python agroinform_scraper.py --output ./agro --headed --start-page 1 --max-pages 1
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-title`: csak adott címrészletet tartalmazó témák feldolgozása
* `--start-page`: kezdő fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali fórumlista oldalak száma
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén

=== Gepigeny scraper

A `gepigeny_scraper.py` a `gepigeny.hu` fórumát dolgozza fel. A scraper először a fórumcsoportokat gyűjti ki, majd az egyes témákból a hozzászólásokat menti JSON formátumban.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny
----

A fenti parancs az eredményeket az `./Gepigeny` mappába menti.

==== Kimeneti mappaszerkezet

A scraper a megadott kimeneti mappán belül saját almappát hoz létre:

[source,text]
----
Gepigeny/
└── gepigeny_forum/
    ├── data/
    │   └── <fórumcsoport>/
    │       └── <topic>.json
    └── state/
        ├── visited_topics.txt
        └── visited_forum_groups.txt
----

A `data` mappába kerülnek a kigyűjtött topic JSON fájlok, a `state` mappa pedig a már feldolgozott témák és fórumcsoportok állapotát tárolja.

==== Opcionális paraméterek

Példa látható böngészővel és komment előnézettel:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny --headed --preview --delay 3
----

Példa csak egy adott fórumcsoport feldolgozására:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny --headed --only-group "Játékokról általában"
----

Hasznosabb kapcsolók:

* `--output`: kimeneti mappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-group`: csak adott fórumcsoport feldolgozása
* `--only-topic`: csak adott topic feldolgozása
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén
* `--auto-reset-fetches`: automatikus böngésző reset adott számú fetch után
* `--preview`: komment előnézet kiírása a terminálba
* `--debug-main-html`: a fő fórumoldal HTML-jének mentése hibakereséshez


=== HardverApró scraper

A `hardverapro_scraper.py` a `hardverapro.hu` apróhirdetéseit gyűjti ki. A scraper a hirdetéslista oldalakat offset alapján járja be, megnyitja az egyes hirdetéseket, majd a hirdetés szövegét, címét, árát, hirdetőjét, dátumát és egyéb metaadatait JSON formátumban menti.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --delay 1 --headless
----

A fenti parancs az eredményeket az `./Hardverapro` mappába menti, azon belül pedig a scraper létrehozza a saját `hardverapro` almappáját.

==== Kimeneti mappaszerkezet

[source,text]
----
Hardverapro/
└── hardverapro/
    ├── hardverapro.json
    └── visited_hardverapro_ads.txt
----

A `hardverapro.json` tartalmazza a kigyűjtött hirdetéseket, a `visited_hardverapro_ads.txt` pedig a már feldolgozott hirdetések azonosítóit tárolja. Ez segít abban, hogy megszakított futás után a scraper ne dolgozza fel újra ugyanazokat a hirdetéseket.

==== Opcionális paraméterek

Példa adott offsetről indítva:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --start-offset 500 --delay 1 --headless
----

Példa biztonságosabb, de lassabb mentéssel:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --delay 1 --headless --fsync
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre a `hardverapro` mappa
* `--start-offset`: kezdő listaoldali offset, csak 100-zal osztható érték lehet, például `0`, `100`, `200`
* `--delay`: várakozás másodpercben az oldalak között
* `--headless`: böngésző futtatása háttérben, látható ablak nélkül
* `--page-timeout`: oldalbetöltési és elemkeresési timeout másodpercben
* `--popup-timeout`: popupok keresésére szánt idő másodpercben
* `--fsync`: minden mentés után fizikai lemezre írja az adatot, biztonságosabb, de lassabb
* `--max-empty-offsets`: ennyi egymás utáni üres listaoldal után áll le a scraper

==== Megjegyzések

A scraper a hirdetéseket egy közös JSON fájl `comments` tömbjébe menti. Ha a `hardverapro.json` már lezárt, befejezett állapotú fájl, akkor a script nem írja felül automatikusan; új futtatáshoz a meglévő `hardverapro.json` és `visited_hardverapro_ads.txt` fájlokat törölni kell.


=== Port scraper

A `port_scraper.py` a `port.hu` fórum témáit és hozzászólásait gyűjti ki. A scraper a fő fórumoldal topiclistáit járja be, majd az egyes témák kommentoldalait dolgozza fel, és a hozzászólásokat topiconként külön JSON fájlba menti.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python port_scraper.py --output ./port
----

A fenti parancs az eredményeket az `./port` mappába menti. A scraper ezen belül létrehozza a saját `port` almappáját.

==== Kimeneti mappaszerkezet

[source,text]
----
port/
└── port/
    ├── topics/
    │   └── <topic>.json
    └── visited.txt
----

A `topics` mappába kerülnek a témánként létrehozott JSON fájlok, a `visited.txt` pedig a már feldolgozott topicok URL-jeit tárolja. Ez segít abban, hogy megszakított vagy újraindított futásnál a scraper ne dolgozza fel újra a már kész témákat.

==== Opcionális paraméterek

Példa látható böngészőablakkal:

[source,bash]
----
python port_scraper.py --output ./port --headed
----

Példa adott főoldali fórumlista oldaltól, limitált oldalszámmal:

[source,bash]
----
python port_scraper.py --output ./port --start-page 1 --max-pages 1
----

Példa csak adott címrészletet tartalmazó topicok feldolgozására:

[source,bash]
----
python port_scraper.py --output ./port --only-title "film"
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre a `port` mappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-title`: csak adott címrészletet tartalmazó topicok feldolgozása
* `--start-page`: kezdő főoldali fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali fórumlista oldalak száma
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén
* `--topic-reset-interval`: ennyi kommentoldalanként teljes böngésző context reset hosszú topicoknál
* `--auto-reset-fetches`: automatikus böngésző context reset adott számú fetch után

==== Megjegyzések

A scraper streamelt JSON-írást használ, vagyis a hozzászólásokat futás közben folyamatosan hozzáfűzi a topic JSON fájljához. Ha egy topic fájl már lezárt JSON-ként létezik, a scraper késznek veszi. Félkész topic fájl esetén az utolsó mentett komment alapján megpróbálja folytatni a feldolgozást.

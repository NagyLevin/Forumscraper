= Scraper projekt
:toc: left
:toclevels: 3
:sectnums:
:source-highlighter: rouge

Ez a projekt több, különálló scraper scriptet tartalmaz. A futtatáshoz ajánlott külön Python virtuális környezetet használni.

== Telepítés

A függőségeket ajánlott `.venv` virtuális környezetbe telepíteni:

[source,bash]
----
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python -m playwright install chromium
----

Windows alatt az aktiválás:

[source,powershell]
----
.venv\Scripts\activate
----

== Általános futtatási megjegyzések

A scraperek általában a `--output` paraméterrel megadott mappába mentik az eredményeket. Hosszabb futásnál érdemes külön kimeneti mappát használni minden scraperhez, hogy az adatok és az állapotfájlok áttekinthetőek maradjanak.

Hibakereséshez több scraper támogatja a `--headed` kapcsolót, amely látható böngészőablakban futtatja a Playwrightot.

== Scraperek

=== Agroinform scraper

A `agroinform_scraper.py` az `agroinform.hu` fórum témáit és hozzászólásait gyűjti ki, majd JSON formátumban menti el a megadott kimeneti mappába.

==== Futtatás

A legfontosabb futtatóparancs:

[source,bash]
----
python agroinform_scraper.py --output ./agro
----

A fenti parancs az eredményeket az `./agro` mappába menti.

==== Opcionális paraméterek

[source,bash]
----
python agroinform_scraper.py --output ./agro --headed --start-page 1 --max-pages 1
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-title`: csak adott címrészletet tartalmazó témák feldolgozása
* `--start-page`: kezdő fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali fórumlista oldalak száma
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén

=== Gepigeny scraper

A `gepigeny_scraper.py` a `gepigeny.hu` fórumát dolgozza fel. A scraper először a fórumcsoportokat gyűjti ki, majd az egyes témákból a hozzászólásokat menti JSON formátumban.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny
----

A fenti parancs az eredményeket az `./Gepigeny` mappába menti.

==== Kimeneti mappaszerkezet

A scraper a megadott kimeneti mappán belül saját almappát hoz létre:

[source,text]
----
Gepigeny/
└── gepigeny_forum/
    ├── data/
    │   └── <fórumcsoport>/
    │       └── <topic>.json
    └── state/
        ├── visited_topics.txt
        └── visited_forum_groups.txt
----

A `data` mappába kerülnek a kigyűjtött topic JSON fájlok, a `state` mappa pedig a már feldolgozott témák és fórumcsoportok állapotát tárolja.

==== Opcionális paraméterek

Példa látható böngészővel és komment előnézettel:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny --headed --preview --delay 3
----

Példa csak egy adott fórumcsoport feldolgozására:

[source,bash]
----
python gepigeny_scraper.py --output ./Gepigeny --headed --only-group "Játékokról általában"
----

Hasznosabb kapcsolók:

* `--output`: kimeneti mappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-group`: csak adott fórumcsoport feldolgozása
* `--only-topic`: csak adott topic feldolgozása
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén
* `--auto-reset-fetches`: automatikus böngésző reset adott számú fetch után
* `--preview`: komment előnézet kiírása a terminálba
* `--debug-main-html`: a fő fórumoldal HTML-jének mentése hibakereséshez


=== HardverApró scraper

A `hardverapro_scraper.py` a `hardverapro.hu` apróhirdetéseit gyűjti ki. A scraper a hirdetéslista oldalakat offset alapján járja be, megnyitja az egyes hirdetéseket, majd a hirdetés szövegét, címét, árát, hirdetőjét, dátumát és egyéb metaadatait JSON formátumban menti.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --delay 1 --headless
----

A fenti parancs az eredményeket az `./Hardverapro` mappába menti, azon belül pedig a scraper létrehozza a saját `hardverapro` almappáját.

==== Kimeneti mappaszerkezet

[source,text]
----
Hardverapro/
└── hardverapro/
    ├── hardverapro.json
    └── visited_hardverapro_ads.txt
----

A `hardverapro.json` tartalmazza a kigyűjtött hirdetéseket, a `visited_hardverapro_ads.txt` pedig a már feldolgozott hirdetések azonosítóit tárolja. Ez segít abban, hogy megszakított futás után a scraper ne dolgozza fel újra ugyanazokat a hirdetéseket.

==== Opcionális paraméterek

Példa adott offsetről indítva:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --start-offset 500 --delay 1 --headless
----

Példa biztonságosabb, de lassabb mentéssel:

[source,bash]
----
python hardverapro_scraper.py --output ./Hardverapro --delay 1 --headless --fsync
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre a `hardverapro` mappa
* `--start-offset`: kezdő listaoldali offset, csak 100-zal osztható érték lehet, például `0`, `100`, `200`
* `--delay`: várakozás másodpercben az oldalak között
* `--headless`: böngésző futtatása háttérben, látható ablak nélkül
* `--page-timeout`: oldalbetöltési és elemkeresési timeout másodpercben
* `--popup-timeout`: popupok keresésére szánt idő másodpercben
* `--fsync`: minden mentés után fizikai lemezre írja az adatot, biztonságosabb, de lassabb
* `--max-empty-offsets`: ennyi egymás utáni üres listaoldal után áll le a scraper

==== Megjegyzések

A scraper a hirdetéseket egy közös JSON fájl `comments` tömbjébe menti. Ha a `hardverapro.json` már lezárt, befejezett állapotú fájl, akkor a script nem írja felül automatikusan; új futtatáshoz a meglévő `hardverapro.json` és `visited_hardverapro_ads.txt` fájlokat törölni kell.


=== Port scraper

A `port_scraper.py` a `port.hu` fórum témáit és hozzászólásait gyűjti ki. A scraper a fő fórumoldal topiclistáit járja be, majd az egyes témák kommentoldalait dolgozza fel, és a hozzászólásokat topiconként külön JSON fájlba menti.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python port_scraper.py --output ./port
----

A fenti parancs az eredményeket az `./port` mappába menti. A scraper ezen belül létrehozza a saját `port` almappáját.

==== Kimeneti mappaszerkezet

[source,text]
----
port/
└── port/
    ├── topics/
    │   └── <topic>.json
    └── visited.txt
----

A `topics` mappába kerülnek a témánként létrehozott JSON fájlok, a `visited.txt` pedig a már feldolgozott topicok URL-jeit tárolja. Ez segít abban, hogy megszakított vagy újraindított futásnál a scraper ne dolgozza fel újra a már kész témákat.

==== Opcionális paraméterek

Példa látható böngészőablakkal:

[source,bash]
----
python port_scraper.py --output ./port --headed
----

Példa adott főoldali fórumlista oldaltól, limitált oldalszámmal:

[source,bash]
----
python port_scraper.py --output ./port --start-page 1 --max-pages 1
----

Példa csak adott címrészletet tartalmazó topicok feldolgozására:

[source,bash]
----
python port_scraper.py --output ./port --only-title "film"
----

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre a `port` mappa
* `--delay`: várakozási idő másodpercben az oldalak betöltése után
* `--only-title`: csak adott címrészletet tartalmazó topicok feldolgozása
* `--start-page`: kezdő főoldali fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali fórumlista oldalak száma
* `--headed`: böngészőablak megjelenítése hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma letöltési hiba esetén
* `--topic-reset-interval`: ennyi kommentoldalanként teljes böngésző context reset hosszú topicoknál
* `--auto-reset-fetches`: automatikus böngésző context reset adott számú fetch után

==== Megjegyzések

A scraper streamelt JSON-írást használ, vagyis a hozzászólásokat futás közben folyamatosan hozzáfűzi a topic JSON fájljához. Ha egy topic fájl már lezárt JSON-ként létezik, a scraper késznek veszi. Félkész topic fájl esetén az utolsó mentett komment alapján megpróbálja folytatni a feldolgozást.

=== PROHARDVER scraper

A `prohardver_scraper.py` a `prohardver.hu` fórum témáit és hozzászólásait gyűjti ki. A script jelenlegi beállítása az `alaplap_chipset_ram` témalistát dolgozza fel, offset alapú lapozással halad végig a topicokon, majd topiconként külön JSON fájlba menti a hozzászólásokat.

Fontos különbség a többi scraperhez képest, hogy ez a scraper Selenium alapú, ezért futtatásához Chrome/Chromium és működő Selenium WebDriver környezet szükséges.

==== Futtatás

Alap futtatóparancs offset tartománnyal:

[source,bash]
----
python prohardver_scraper.py 0 6000 --output . --delay 3 --headless
----

A két kötelező pozicionális paraméter a kezdő és a záró offset. Az offseteknek 100-zal osztható értékeknek kell lenniük, például `0`, `100`, `200`, `6000`.

Példa kisebb tesztfutásra:

[source,bash]
----
python prohardver_scraper.py 0 200 --output ./Prohardver --delay 2 --headless
----

==== Kimeneti mappaszerkezet

[source,text]
----
Prohardver/
└── prohardver/
    ├── alaplap_chipset_ram/
    │   └── <topic>.json
    └── visited_alaplap_chipset_ram.txt
----

Az `alaplap_chipset_ram` mappába kerülnek a topiconként létrehozott JSON fájlok, a `visited_alaplap_chipset_ram.txt` pedig a már teljesen feldolgozott topicok URL-jeit tárolja.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `start_offset`: kötelező kezdő offset, például `0`
* `end_offset`: kötelező záró offset, például `6000`
* `--output`: kimeneti alapmappa, ezen belül jön létre a `prohardver` mappa
* `--delay`: várakozás másodpercben az oldalak között
* `--headless`: böngésző futtatása háttérben, látható ablak nélkül

==== Megjegyzések

A scraper megszakított futás után képes folytatni a feldolgozást. Félkész topic JSON esetén az utolsó mentett komment vagy resume URL alapján próbál továbbhaladni, teljesen lezárt JSON esetén pedig a topicot késznek veszi.

A script a hozzászólásokat futás közben folyamatosan hozzáfűzi a JSON fájlhoz, majd a topic végén lezárja a fájlt. Ha egy topic hibával megáll, nem kerül be a visited listába, így később újra próbálható.


=== Vatera scraper

A `vatera_scraper.py` a `vatera.hu` hirdetéslistáit dolgozza fel. Alapértelmezés szerint a műszaki cikk és mobil kategóriából indul, de a `--url` paraméterrel tetszőleges Vatera listaoldal megadható. A scraper Playwright + BeautifulSoup alapon működik, a hirdetéskártyákból kigyűjti a termékoldalak URL-jeit, majd az egyes hirdetésekről címet, eladót, dátumot és leírást ment JSON formátumba.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python vatera_scraper.py --out ./vatera --topic muszaki --preview --delay 3
----

A fenti parancs az eredményeket az `./vatera` kimeneti mappába menti, azon belül pedig a scraper létrehozza a saját `vatera` almappáját.

Példa csak az első 3 oldal tesztelésére, látható böngészőablakkal:

[source,bash]
----
python vatera_scraper.py --out ./vatera --topic muszaki_teszt --start-page 1 --end-page 3 --preview --headed
----

Példa másik Vatera lista URL megadásával:

[source,bash]
----
python vatera_scraper.py --url "https://www.vatera.hu/muszaki-cikk-es-mobil/index-c12082.html" --out ./vatera --topic vatera_muszaki --preview
----

Példa gyorsabb, headless futáshoz nehezebb erőforrások tiltásával:

[source,bash]
----
python vatera_scraper.py --out ./vatera --topic muszaki --preview --delay 2 --block-heavy
----

==== Kimeneti mappaszerkezet

[source,text]
----
vatera/
└── vatera/
    ├── topics/
    │   └── <topic>.json
    └── state/
        └── visited_ads.txt
----

A `topics` mappába kerül a megadott `--topic` név alapján létrehozott JSON fájl, a `state/visited_ads.txt` pedig a már feldolgozott hirdetések URL-jeit tárolja. Ez segít abban, hogy újraindított futásnál a scraper ne mentse újra ugyanazokat a hirdetéseket.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `--url`: a feldolgozandó Vatera listaoldal URL-je
* `--topic`: a mentett JSON fájl neve kiterjesztés nélkül
* `--out`, `--output`: kimeneti mappa, ezen belül jön létre a `vatera` mappa
* `--delay`: várakozás másodpercben a lekérések között
* `--start-page`: kezdő listaoldal
* `--end-page`: utolsó feldolgozandó listaoldal
* `--max-pages`: biztonsági maximum, ha az oldalszám nem állapítható meg
* `--preview`: eladó, dátum és rövid leírás előnézetének kiírása a terminálba
* `--headed`: látható böngészőablak hibakereséshez
* `--timeout-ms`: timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma fetch hiba esetén
* `--auto-reset-fetches`: ennyi fetch után újranyitja a browser contextet
* `--block-heavy`: képek, fontok és média tiltása gyorsításhoz

==== Megjegyzések

A scraper streamelt JSON append módban menti a hirdetéseket, vagyis futás közben folyamatosan hozzáfűzi az új elemeket a JSON fájlhoz. Ha a topic JSON már létezik és le van zárva, a script újranyitja hozzáfűzéshez, majd a futás végén ismét lezárja.

Hosszabb Vatera futásoknál érdemes nagyobb `--delay` értéket használni, például 2-5 másodpercet, és szükség esetén `--start-page` / `--end-page` paraméterekkel kisebb tartományokra bontani a feldolgozást.

=== Index Fórum scraper

Az `index_scraper.py` az Index Fórum főoldaláról indulva bejárja a nagy fórumcsoportokat, azon belül az alforumokat, majd a topicok hozzászólásait menti topiconként külön JSON fájlokba. A scraper Playwright + BeautifulSoup alapon működik, és kezeli a visited állapotfájlokat, hogy újraindítás után ne dolgozza fel újra a már kész topicokat.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python index_scraper.py --output ./index --delay 2
----

Példa csak egy adott nagy kategóriára:

[source,bash]
----
python index_scraper.py --output ./index --only-category "Általános" --delay 2
----

Példa csak egy adott alforumra:

[source,bash]
----
python index_scraper.py --output ./index --only-subforum "A törzsasztal" --delay 2
----

==== Kimeneti mappaszerkezet

[source,text]
----
index/
└── index/
    ├── <fórumcsoport>/
    │   └── <alforum>/
    │       └── <topic>.json
    ├── visited_topics.txt
    └── visited_forumgroups.txt
----

A `visited_topics.txt` a feldolgozott topicokat, a `visited_forumgroups.txt` pedig a teljesen feldolgozott alforumokat tárolja.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre az `index` mappa
* `--delay`: várakozás másodpercben az oldalak között
* `--only-category`: csak adott nagy fórumcsoport feldolgozása
* `--only-subforum`: csak adott alforum feldolgozása
* `--headed`: látható böngészőablak hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma fetch hiba esetén
* `--auto-reset-fetches`: automatikus browser context reset adott számú fetch után
* `--topic-reset-interval`: ennyi kommentoldalanként teljes context reset hosszú topicoknál
* `--subforum-reset-interval`: ennyi topiclista oldalanként teljes context reset alforumon belül

==== Megjegyzések

A scraper appendelt JSON mentést használ. Félkész topic fájl esetén az utolsó mentett komment URL-je alapján próbálja folytatni a feldolgozást. A főoldal HTML-je hibakereséshez `debug_index_main.html` néven is mentésre kerülhet a kimeneti alapmappába.

        └── visited_categories.txt
----

A `data` mappában a szekció/kategória/topic bontás szerinti JSON fájlok vannak. A `state` mappa tárolja a már feldolgozott témákat és kategóriákat.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `--output`: kimeneti mappa
* `--delay`: várakozás másodpercben az oldalak között
* `--only-section`: csak adott szekció feldolgozása
* `--only-category`: csak adott kategória feldolgozása
* `--only-topic`: csak adott téma feldolgozása
* `--headed`: látható böngészőablak hibakereséshez
* `--timeout-ms`: timeout ezredmásodpercben
* `--retries`: újrapróbálkozások száma
* `--topic-reset-interval`: context reset intervallum hosszú topicoknál
* `--auto-reset-fetches`: automatikus context reset adott számú fetch után
* `--preview`: komment előnézet kiírása
* `--debug-head`: a fő fórumoldal HTML head részének kiírása hibakereséshez

==== Megjegyzések

A scraper csak a konfigurált fórumrészeket dolgozza fel, például az általános, számítástechnikai és hírekhez kapcsolódó fórumokat. A topic JSON fájlok streamelve készülnek, a félkész fájlokból pedig újraindításkor folytatható a mentés.


=== Hobbielektronika scraper

A `hobbye_scraper.py` a `hobbielektronika.hu` fórum főoldali topiclistáit járja be, majd topiconként menti a hozzászólásokat. A scraper Playwright + BeautifulSoup alapon működik, és külön figyeli a szerver túlterheltségi tiltóoldalát.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python hobbye_scraper.py --output ./hobbielektronika
----

Példa látható böngészőablakkal:

[source,bash]
----
python hobbye_scraper.py --output ./hobbielektronika --headed
----

Példa adott címrészletre és oldallimitre:

[source,bash]
----
python hobbye_scraper.py --output ./hobbielektronika --only-title "arduino" --start-page 1 --max-pages 3 --headed
----

==== Kimeneti mappaszerkezet

[source,text]
----
hobbielektronika/
└── hobbielektronika/
    ├── topics/
    │   └── <topic>.json
    └── visited.txt
----

A `topics` mappába kerülnek a topiconkénti JSON fájlok, a `visited.txt` pedig a már feldolgozott topicok URL-jeit tárolja.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa
* `--delay`: várakozás másodpercben az oldalak között
* `--only-title`: csak adott címrészletet tartalmazó topicok feldolgozása
* `--start-page`: kezdő főoldali fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali listaoldalak száma
* `--headed`: látható böngészőablak hibakereséshez
* `--fetch-retries`: fetch újrapróbálkozások száma
* `--retry-sleep`: várakozás az újrapróbálások között
* `--nav-timeout-ms`: navigációs timeout ezredmásodpercben
* `--default-timeout-ms`: általános Playwright timeout ezredmásodpercben

==== Megjegyzések

Ha a szerver excessive load policy oldalt ad vissza, a scraper külön hibával leáll, hogy ne terhelje tovább az oldalt. A topic JSON fájlok streamelt append módban készülnek, és visited fájl segíti az újraindíthatóságot.


=== Hoxa scraper

A `hoxa_scraper.py` a `hoxa.hu` fórumlistáit és topicjait dolgozza fel. A scraper kezeli a Hoxa egyedi fórum URL-struktúráját, oldallapozását, a cookie/consent rétegeket, valamint CAPTCHA észlelésekor leáll.

==== Futtatás

Alap futtatóparancs:

[source,bash]
----
python3 hoxa_scraper.py --output ./hoxa
----

Példa oldaltartomány feldolgozására látható böngészővel:

[source,bash]
----
python3 hoxa_scraper.py --output ./hoxa --start-page 1 --end-page 10 --max-pages 10 --headed
----

Példa csak adott címrészletű topicokra:

[source,bash]
----
python3 hoxa_scraper.py --output ./hoxa --only-title "segély" --headed
----

==== Kimeneti mappaszerkezet

[source,text]
----
hoxa/
└── hoxa/
    ├── topics/
    │   └── <topic>.json
    └── visited.txt
----

A `topics` mappába kerülnek a topiconkénti JSON fájlok, a `visited.txt` pedig a már feldolgozott topicok URL-jeit tárolja.

==== Opcionális paraméterek

Hasznosabb kapcsolók:

* `--output`: kimeneti alapmappa, ezen belül jön létre a `hoxa` mappa
* `--delay`: várakozás másodpercben az oldalak között
* `--only-title`: csak adott címrészletet tartalmazó topicok feldolgozása
* `--start-page`: kezdő fórumlista oldal
* `--end-page`: utolsó feldolgozandó fórumlista oldal
* `--max-pages`: maximum feldolgozandó főoldali listaoldalak száma
* `--headed`: látható böngészőablak hibakereséshez
* `--timeout-ms`: navigációs timeout ezredmásodpercben
* `--retries`: fetch újrapróbálkozások száma
* `--topic-reset-interval`: ennyi kommentoldalanként teljes context reset hosszú topicoknál
* `--auto-reset-fetches`: ennyi fetch után automatikus context reset

==== Megjegyzések

A scraper streamelt JSON append módban menti a hozzászólásokat, és félkész topic fájl esetén az utolsó mentett komment alapján próbál folytatni. CAPTCHA észlelésekor a futás hibával megáll, ilyenkor érdemes később, kisebb oldaltartománnyal és nagyobb `--delay` értékkel újrapróbálni.
